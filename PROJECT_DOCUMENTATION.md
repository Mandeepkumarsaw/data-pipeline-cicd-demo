# 1. Overview

This project implements a complete CI/CD workflow for an AWS Lambda ETL application.

The objective is to:

* Store source code in GitHub
* Automatically trigger builds on code changes
* Package Lambda source code into a deployment ZIP
* Store artifacts in S3
* Deploy the latest package to AWS Lambda
* Eliminate manual deployment effort

### AWS Services Used

* AWS Lambda
* AWS CodePipeline
* AWS CodeBuild
* Amazon S3
* GitHub
* IAM
* CloudWatch Logs

# 7. Problems Encountered

## Issue 1: BuildOutput Folder Missing

### Observation

Pipeline showed:

```text
Source = Success
Build = Success
```

But:

```text
BuildOutput folder not found
function.zip not found
```

### Root Cause

CodeBuild project was incorrectly configured.

Source configuration:

```text
No Source
```

with incorrect artifact handling.

---

## Issue 2: Unable to Locate ZIP File

Expected:

```text
BuildOutput/
   └── function.zip
```

Actual:

```text
Only random artifact files visible
```

### Investigation

Build logs were checked.

Artifact upload completed successfully but packaging configuration was incorrect.

---

## Issue 3: buildspec.yml Location Confusion

Initially assumed:

```text
lambda/etl_customer/buildspec.yml
```

Actual location:

```text
Repository Root

data-pipeline-cicd-demo/
└── buildspec.yml
```

### Resolution

Confirmed buildspec.yml belongs in repository root.

---

## Issue 4: Incorrect CodeBuild Configuration

Old CodeBuild project:

```text
etl-customer-build
```

was created with:

```text
Source Provider = No Source
```

and configuration mismatch.

### Resolution

Deleted and recreated the build project.

---

## Issue 5: Source Configuration Confusion

While recreating CodeBuild:

User attempted:

```text
GitHub
Amazon S3
No Source
```

### Resolution

Since CodePipeline already supplies source artifacts:

```text
Source Provider = No Source
```

is correct.

CodePipeline passes SourceOutput automatically.

# 9. BuildSpec Configuration

## Final Working BuildSpec

```yaml
version: 0.2

phases:
  build:
    commands:
      - echo "Packaging Lambda..."
      - cd lambda/etl_customer
      - zip -r ../../function.zip .

artifacts:
  files:
    - function.zip
```

---

## Build Process

Step 1

```bash
cd lambda/etl_customer
```

Move to Lambda source folder.

Step 2

```bash
zip -r ../../function.zip .
```

Create deployment package.

Step 3

```yaml
artifacts:
  files:
    - function.zip
```

Export ZIP to CodePipeline.

# 15. Lessons Learned

* CodePipeline supplies source artifacts to CodeBuild.
* buildspec.yml should remain in repository root.
* BuildOutput exists only after successful artifact creation.
* Lambda deployment package must contain all required files.
* Deploy stage removes need for manual Lambda uploads.
* CodeBuild source configuration can be "No Source" when integrated with CodePipeline.
* Build logs are the fastest way to troubleshoot packaging issues.
